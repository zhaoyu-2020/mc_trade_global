# global-trade-website 外贸企业官网项目需求文档（完整版）

**文档版本**：V2.0

**文档状态**：定稿（可直接用于开发、测试、验收）

**适用对象**：项目经理、前端开发、后端开发、测试工程师、产品运营、企业管理员

**项目名称**mc_trade_global（GitHub 仓库名）

**项目类型**：外贸企业宣传官网（前后端分离架构）

**核心目标**：搭建符合海外客户浏览习惯、可快速展示企业实力与产品的宣传官网；提供可视化后台管理系统，支持非技术人员快速修改官网所有内容（菜单、轮播图、文章、产品等），降低运营维护成本，助力企业海外获客。

# 一、项目概述

## 1.1 项目背景

为满足外贸企业品牌出海、产品展示、海外客户对接需求，解决传统官网修改内容需依赖技术人员、部署繁琐的问题，搭建一套前后端分离的外贸宣传官网，实现“后台可视化操作、前端实时同步更新”，适配海外访问场景，提升企业海外品牌形象与客户转化效率。

## 1.2 项目范围

本项目包含两大核心模块：前端用户端（外贸官网）、后端管理端（内容管理系统），涵盖内容展示、内容管理、权限控制、数据统计、系统配置等全流程功能，不包含在线交易、支付、会员体系（可作为后续扩展功能）。

## 1.3 项目架构

- **前端架构**：采用主流前端框架，实现响应式布局，适配 PC 端、平板端、移动端，支持多语言切换，静态资源（图片、文档）采用 CDN 加速，保障海外访问速度。

- **后端架构**：采用前后端分离模式，提供 RESTful API 接口，负责数据存储、业务逻辑处理、权限校验，支持接口加密、防注入、防盗链等安全措施。

- **数据交互**：后台管理端修改内容后，通过 API 接口同步至前端，前端无需重新打包部署，实时展示最新内容；前端用户提交的留言、咨询等数据，实时同步至后台，供管理员查看处理。

- **数据库**：采用关系型数据库，存储用户数据、产品数据、文章数据、配置数据等，支持数据备份与恢复。

## 1.4 运行环境

### 1.4.1 前端运行环境

- 开发环境：Node.js（v16+）、npm/yarn（最新稳定版）
- 技术栈：vue3，element-plus

- 运行环境：主流浏览器（Chrome 90+、Firefox 88+、Edge 90+、Safari 14+），移动端浏览器（微信浏览器、Chrome 移动端、Safari 移动端）

- 部署环境：静态服务器（Nginx）、CDN 加速

### 1.4.2 后端运行环境

- 开发环境：JDK 17 / Node.js v16+（若用 Node.js）、开发工具（IDEA/Vscode）
- 技术栈：springboot，mysql

- 运行环境：服务器（Linux CentOS 7+）、数据库（MySQL 8.0+）、容器（Docker 可选）

- 部署环境：Tomcat 9+（Java）/ Nginx + PM2（Node.js）

# 二、总体功能需求

本项目功能分为两大模块：前端用户端（官网）、后端管理端（内容管理系统），两大模块相互关联，后台管理内容，前端展示内容，具体功能如下：

|模块|核心功能|目标用户|
|---|---|---|
|前端用户端|多语言浏览、页面展示、产品查看、文章阅读、在线留言、导航跳转|海外客户、潜在访客、合作伙伴|
|后端管理端|账号权限管理、导航菜单配置、轮播图管理、页面内容编辑、产品管理、文章管理、留言管理、系统设置、数据统计、数据备份|企业超级管理员、普通管理员、运营人员|
# 三、详细功能需求（前端用户端）

## 3.1 通用模块（全页面通用）

### 3.1.1 响应式布局

- 适配分辨率：PC 端（1920×1080、1366×768、1440×900）、平板端（iPad Pro 10.5 英寸、iPad Air 9.7 英寸）、移动端（375×667、414×736、360×640）。

- 适配规则：PC 端展示完整布局，移动端隐藏非核心内容（如侧边栏）、导航菜单转为汉堡菜单，图片、文字自动缩放，避免出现横向滚动条。

- 交互要求：移动端点击汉堡菜单弹出导航，点击空白处关闭；下拉时顶部导航栏固定（可选），提升浏览体验。

### 3.1.2 多语言支持

- 默认语言：中文（简体）、英文（必填），支持后续扩展其他语言（如西班牙语、法语、俄语）。

- 切换功能：顶部导航栏右侧设置语言切换按钮（中文/英文），点击切换后，全页面文字、菜单、按钮同步切换，无需刷新页面。

- 语言配置：所有语言文字均由后台统一配置，支持后台修改各语言对应的文案，前端实时同步。

### 3.1.3 顶部导航栏

- 展示内容：后台配置的一级菜单、二级菜单，语言切换按钮，企业 LOGO（后台可上传替换）。

- 交互规则：
        

    - 一级菜单 hover 时，显示对应的二级菜单（若有），二级菜单点击跳转至对应页面。

    - 当前页面对应的菜单高亮显示，方便用户定位当前位置。

    - 移动端导航栏：LOGO 居左，汉堡菜单居右，点击汉堡菜单弹出所有菜单（一级+二级），支持滚动查看。

- 异常处理：菜单跳转链接无效时，显示“页面不存在”提示，并提供返回首页按钮。

### 3.1.4 页脚

- 展示内容：企业版权信息（格式：© 年份 企业名称 保留所有权利）、企业地址、联系电话、邮箱、官网地址、社交媒体链接（Facebook、LinkedIn、Twitter 等，后台可配置显示/隐藏及链接）、网站备案信息（若有）。

- 交互规则：社交媒体链接点击跳转至对应平台官方页面（新窗口打开）；邮箱点击自动唤起本地邮件客户端，填写收件人地址；电话点击（移动端）唤起拨号功能。

- 样式要求：页脚背景色与官网整体风格一致，文字居中（移动端）/ 左右分布（PC 端），字体大小适中，清晰易读。

### 3.1.5 加载状态与异常提示

- 加载状态：页面、图片、数据加载时，显示加载动画（如旋转图标），加载完成后自动隐藏，避免用户误以为页面卡死。

- 异常提示：
        

    - 网络异常：显示“网络连接失败，请检查网络后重试”，提供“刷新页面”按钮。

    - 页面不存在：显示“404 页面不存在”，提供“返回首页”按钮。

    - 数据加载失败：显示“数据加载失败，请重试”，提供“重试”按钮。

### 3.1.6 图片懒加载

所有页面图片（轮播图、产品图、详情图等）均支持懒加载，即用户滚动到图片所在位置时再加载，提升页面打开速度，降低海外访问带宽压力。

## 3.2 首页模块

### 3.2.1 全屏轮播大图

- 展示位置：首页顶部，占满屏幕宽度，高度适配不同设备（PC 端约 500px，移动端约 250px）。

- 展示内容：后台上传的轮播图片、标题、描述（均支持多语言），跳转链接（可选）。

- 交互规则：
        

    - 自动轮播：默认每 5 秒切换一张图片，轮播过程有过渡动画（如淡入淡出）。

    - 手动切换：PC 端鼠标点击轮播图两侧箭头，移动端滑动屏幕，可切换上下张图片；轮播图下方显示指示器（小圆点），点击指示器可直接切换至对应图片，当前图片对应的指示器高亮。

    - 跳转功能：点击轮播图（或标题/描述），跳转至后台配置的链接（可跳转至产品详情、文章详情、外部链接等），新窗口/当前窗口打开可后台配置。

    - 暂停规则：鼠标 hover 轮播图（PC 端）、手指按住轮播图（移动端），轮播暂停，离开后继续轮播。

- 异常处理：轮播图片加载失败时，显示默认占位图，不影响整体页面布局。

### 3.2.2 企业简介

- 展示位置：轮播图下方，居中展示。

- 展示内容：企业名称（多语言）、简介文案（多语言，支持换行、分段）、企业形象图（1-2 张，后台可上传替换）。

- 样式要求：文案左对齐（PC 端）/ 居中（移动端），字体清晰，图片与文案布局协调，避免拥挤。

- 交互规则：点击“查看更多”按钮（后台可配置显示/隐藏），跳转至“关于我们”页面。

### 3.2.3 企业优势

- 展示位置：企业简介下方，采用卡片式布局（3-6 个优势，后台可配置数量）。

- 展示内容：每个优势卡片包含图标（后台可上传）、优势标题（多语言）、优势描述（多语言）。

- 样式要求：卡片均匀分布（PC 端一行 3 个，移动端一行 1 个），卡片 hover 时有轻微缩放/阴影效果，提升交互体验。

- 配置规则：优势的顺序、图标、标题、描述均由后台配置，支持新增、编辑、删除、排序。

### 3.2.4 核心产品展示

- 展示位置：企业优势下方，包含“产品推荐”标题（多语言，后台可修改）。

- 展示内容：后台标记为“推荐”的产品（数量可后台配置，默认 4-6 个），每个产品展示：产品图片、产品名称（多语言）、产品简短描述（多语言，1-2 行）。

- 交互规则：
        

    - 点击产品卡片，跳转至该产品的详情页面。

    - PC 端支持左右滑动查看更多推荐产品（若数量超过当前展示数量），移动端支持横向滑动。

    - 点击“查看全部产品”按钮，跳转至“产品中心”页面。

- 样式要求：产品卡片均匀分布，图片尺寸统一，文字居中，hover 时有轻微阴影效果。

### 3.2.5 新闻/动态列表

- 展示位置：产品展示下方，包含“最新动态”/“新闻资讯”标题（多语言，后台可修改）。

- 展示内容：后台发布的最新文章（数量可后台配置，默认 3-5 篇），每篇文章展示：文章封面图（可选）、文章标题（多语言）、发布时间、简短摘要（多语言，1 行）。

- 交互规则：
        

    - 点击文章标题/封面图，跳转至文章详情页面。

    - 点击“查看更多新闻”按钮，跳转至“新闻资讯”页面。

    - 文章按发布时间倒序排列（最新发布在前）。

- 样式要求：文章列表左对齐，封面图与文字布局协调，发布时间字体略小，与标题区分开。

### 3.2.6 在线咨询/留言

- 展示位置：首页底部（页脚上方），采用表单形式。

- 表单字段（均为必填项，后台可配置是否必填）：
        

    - 姓名（输入框，提示文字：请输入您的姓名，多语言）

    - 邮箱（输入框，提示文字：请输入您的邮箱，多语言，校验邮箱格式）

    - 电话（输入框，提示文字：请输入您的电话，多语言，支持国际区号输入）

    - 留言内容（文本域，提示文字：请输入您的留言/需求，多语言，限制输入字数 10-500 字）

- 交互规则：
       

    - 点击“提交”按钮，先校验表单字段（必填项是否填写、邮箱格式是否正确、字数是否符合要求），校验通过后提交数据，显示“提交成功，我们将尽快与您联系”提示；校验失败则显示对应错误提示（如“请输入正确的邮箱格式”）。

    - 提交成功后，表单自动清空，提示信息 3 秒后自动消失。

    - 表单提交过程中，按钮禁用，显示加载状态，防止重复提交。

- 样式要求：表单居中展示，输入框、文本域样式统一，按钮样式突出（与页面主色调一致），错误提示文字为红色，清晰可见。

## 3.3 关于我们页面

### 3.3.1 页面结构

页面顶部为标题（多语言，后台可修改），下方分为多个可配置模块（后台可调整模块顺序、显示/隐藏），默认模块：企业介绍、发展历程、企业文化、工厂/办公环境。

### 3.3.2 企业介绍

- 展示内容：富文本内容（多语言），支持插入图片、文字、段落、表格、链接，可设置字体大小、颜色、对齐方式。

- 样式要求：内容居中（移动端）/ 左对齐（PC 端），段落间距适中，图片自适应页面宽度，避免过大或过小。

### 3.3.3 发展历程

- 展示内容：时间轴形式，每个时间节点包含：年份、事件描述（多语言）、事件图片（可选，后台可上传）。

- 交互规则：时间轴支持上下滚动查看，PC 端鼠标 hover 时间节点，显示详细描述（若有）；移动端点击时间节点，展开详细描述。

- 配置规则：时间节点的顺序、年份、描述、图片均由后台配置，支持新增、编辑、删除、排序。

### 3.3.4 企业文化

- 展示内容：核心价值观、企业使命、企业愿景（多语言，后台可编辑），可搭配相关图片（后台可上传）。

- 样式要求：采用卡片式或列表式布局，文字清晰，图片与文字协调。

### 3.3.5 工厂/办公环境

- 展示内容：后台上传的环境图片（多张，支持批量上传），采用图片画廊形式。

- 交互规则：点击图片，弹出大图预览，支持左右切换图片、关闭预览；PC 端支持鼠标滚轮缩放预览图。

- 样式要求：图片均匀分布，尺寸统一，预览图清晰，加载速度快。

## 3.4 产品中心页面

### 3.4.1 页面结构

页面顶部为标题（多语言，后台可修改），左侧为产品分类导航（后台可配置），右侧为产品列表，顶部可设置产品搜索框。

### 3.4.2 产品分类导航

- 展示内容：后台配置的产品分类（一级分类，支持二级分类，若有），分类名称为多语言。

- 交互规则：
        

    - 点击分类，右侧产品列表显示该分类下的所有产品（默认显示全部分类产品）。

    - 当前选中的分类高亮显示，二级分类嵌套在一级分类下方，点击一级分类展开/收起二级分类（PC 端）；移动端分类导航转为下拉菜单。

- 样式要求：分类导航固定在左侧（PC 端），宽度适中，文字清晰；移动端下拉菜单样式与整体风格一致。

### 3.4.3 产品搜索

- 展示位置：产品分类导航上方（PC 端）、页面顶部（移动端）。

- 交互规则：
        

    - 用户输入关键词（支持中英文），点击搜索按钮或按下回车键，产品列表显示包含该关键词的产品（匹配产品名称、描述、型号）。

    - 搜索为空时，显示“未找到相关产品，请更换关键词重试”提示。

    - 支持搜索历史记录（仅本地存储，关闭浏览器后清空），点击历史记录可快速重新搜索。

- 样式要求：搜索框与页面风格一致，搜索按钮突出，提示文字清晰。

### 3.4.4 产品列表

- 展示内容：当前分类/搜索结果下的所有产品，每个产品卡片包含：产品主图、产品名称（多语言）、产品型号（多语言）、产品简短描述（多语言）、是否推荐标记（可选）。

- 交互规则：


    - 点击产品卡片，跳转至该产品的详情页面。

    - 支持分页功能：每页显示产品数量可后台配置（默认 8-12 个），点击页码或“上一页”“下一页”切换页面；无下一页时，“下一页”按钮禁用。

    - 支持排序功能：可按发布时间（默认，最新在前）、产品名称（字母顺序）排序（后台可配置是否显示排序功能）。

- 样式要求：产品卡片均匀分布（PC 端一行 3-4 个，移动端一行 1-2 个），图片尺寸统一，文字居中，hover 时有轻微阴影效果；分页按钮样式清晰，当前页码高亮。

### 3.4.5 产品详情页面

- 页面结构：产品图片展示区、产品基本信息区、产品详细参数区、产品详情描述区、相关产品推荐区。

- 产品图片展示区：
        

    - 展示产品主图+多张详情图（后台上传），主图居中显示，详情图在主图下方（PC 端）/ 右侧（移动端）。

    - 交互规则：点击详情图，切换为主图；点击主图，弹出大图预览，支持左右切换、缩放、关闭。

- 产品基本信息区：
        

    - 展示内容：产品名称（多语言）、产品型号（多语言）、产品价格（可选，后台可配置显示/隐藏及价格内容）、产品库存（可选）、所属分类。
