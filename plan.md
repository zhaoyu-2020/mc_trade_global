# mc_trade_global 外贸企业官网项目开发计划

## 项目概述

- **项目名称**：mc_trade_global
- **项目类型**：外贸企业宣传官网（前后端分离架构）
- **技术栈**：Vue3 + Element Plus（前端）/ Spring Boot + MySQL（后端）
- **项目结构**：2个项目（前端 + 后端），前端通过路由区分用户端和管理端
- **核心目标**：搭建符合海外客户浏览习惯、可快速展示企业实力与产品的宣传官网

## 一、项目架构

```
┌─────────────────────────────────────────────────────────┐
│                   前端（Vue3 + Element Plus）              │
│  ┌─────────────────┐    ┌─────────────────┐             │
│  │  用户端路由 /    │    │  管理端路由 /admin │           │
│  │  首页/产品/新闻等 │    │  仪表盘/菜单/轮播图等 │         │
│  └─────────────────┘    └─────────────────┘             │
└────────────────────────┬────────────────────────────────┘
                         │ RESTful API
┌────────────────────────▼────────────────────────────────┐
│                后端（Spring Boot + MySQL）                │
│  ┌─────────────────┐    ┌─────────────────┐             │
│  │  /api/web/*    │    │  /api/admin/*   │             │
│  │  用户端公开接口   │    │  管理端需鉴权接口  │           │
│  └─────────────────┘    └─────────────────┘             │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│                      MySQL 数据库                        │
└─────────────────────────────────────────────────────────┘
```

## 四、API 接口设计概要

### 用户端接口（无需鉴权）
- `GET /api/web/banner` - 获取轮播图列表
- `GET /api/web/product/recommend` - 获取推荐产品
- `GET /api/web/product/list` - 产品列表（含分类、搜索、分页）
- `GET /api/web/product/:id` - 产品详情
- `GET /api/web/article/latest` - 获取最新文章
- `GET /api/web/article/list` - 文章列表（含分类、搜索、分页）
- `GET /api/web/article/:id` - 文章详情
- `GET /api/web/about` - 获取关于我们内容
- `GET /api/web/contact` - 获取联系方式
- `POST /api/web/message` - 提交留言
- `GET /api/web/menu` - 获取导航菜单

### 管理端接口（需 Token 鉴权）
- `POST /api/admin/auth/login` - 管理员登录
- `POST /api/admin/auth/logout` - 退出登录
- `GET /api/admin/auth/currentUser` - 获取当前用户信息
- `GET/POST/PUT/DELETE /api/admin/menu` - 导航菜单 CRUD
- `GET/POST/PUT/DELETE /api/admin/banner` - 轮播图 CRUD
- `GET/POST/PUT/DELETE /api/admin/product` - 产品 CRUD
- `GET/POST/PUT/DELETE /api/admin/product/category` - 产品分类 CRUD
- `GET/POST/PUT/DELETE /api/admin/article` - 文章 CRUD
- `GET/POST/PUT/DELETE /api/admin/article/category` - 文章分类 CRUD
- `GET/PUT /api/admin/message` - 留言查询/回复
- `GET/PUT /api/admin/config` - 系统配置
- `GET /api/admin/dashboard/stats` - 仪表盘统计数据

## 五、开发规范

### 代码规范
- 前端：Vue3 Composition API、TypeScript（可选）、ESLint + Prettier
- 后端：Spring Boot 编码规范、RESTful API 设计原则
- Git：分支命名（feature/xxx、bugfix/xxx）、Commit 规范

### 命名规范
- 数据库：snake_case（小写下划线）
- 后端：Java 驼峰命名（camelCase）
- 前端：Vue 组件 PascalCase、变量 camelCase

### 安全规范
- 敏感信息加密存储（密码 BCrypt）
- SQL 注入防护（参数化查询）
- XSS 防护（输入过滤、输出编码）
- 接口限流与防刷

## 七、注意事项

1. **多语言支持**：所有前端展示内容均需支持中英文，后端存储多语言字段
2. **响应式适配**：重点测试移动端浏览体验，确保无横向滚动
3. **图片优化**：使用懒加载、WebP 格式、图片压缩
4. **SEO 优化**：静态页面预渲染、Sitemap 生成
5. **CDN 加速**：静态资源走 CDN，提升海外访问速度
